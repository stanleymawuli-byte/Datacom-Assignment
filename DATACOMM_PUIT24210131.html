Introduction to Data Communication

Data communication is the exchange of data between two devices via a transmission medium using an agreed set of rules called a protocol. Five key components make up every communication system:

  • Message — the information (text, number, image, audio, video) being communicated
  • Sender — the device that sends the data message (computer, phone, camera)
  • Receiver — the device that receives the message
  • Transmission Medium — the physical path data travels (wire, fiber, radio)
  • Protocol — the rules governing data communication (TCP/IP, HTTP, SMTP)

Telegraph & Morse Code

⚡ The electric telegraph was the world's first electrical long-distance communication system — the direct ancestor of all modern digital networks.

Historical Timeline

1753
Scottish scientist Charles Morrison proposes electric telegraph using separate wires for each letter
1820s
Hans Christian Ørsted discovers electromagnetism — the scientific foundation for telegraphs
1837
Samuel Morse and Alfred Vail patent the first practical electric telegraph in the United States
1844
First long-distance message sent: "What hath God wrought" — Washington D.C. to Baltimore (64 km) by Samuel Morse
1851
First undersea telegraph cable laid across the English Channel between England and France
1866
Transatlantic telegraph cable successfully completed — connecting Europe and North America
1876
Alexander Graham Bell invents the telephone — beginning the era of voice communication
1920s
Commercial telegraph services peak; Morse code becomes the universal language of radio operators
1999
France is last major country to drop Morse code as a maritime distress requirement

How the Telegraph Worked

An operator pressed a telegraph key to complete an electrical circuit, sending pulses of current down a wire. Short pulses were dots (·) and long pulses were dashes (–). The receiver heard clicks or saw marks on paper tape. This is the original binary signalling system — the same on/off principle used in modern digital computers.

TCP/IP Protocol Suite

TCP/IP is the practical protocol stack used by the internet. It has 4 layers (compared to OSI's 7), mapping roughly to the OSI model.

04

Protocol Data Unit (PDU)

A PDU is the specific block of data used by each OSI layer. As data moves down the layers from sender to receiver, each layer adds its own header (encapsulation). The receiver strips these off going back up (de-encapsulation).

PDU Names per Layer

OSI LayerPDU NameWhat it contains
Application / Presentation / Session (7–5)DataUser data / application payload
Transport (4)Segment (TCP) / Datagram (UDP)Port numbers, sequence numbers
Network (3)PacketSource & destination IP addresses
Data Link (2)FrameMAC addresses, error checking (CRC)
Physical (1)Bits0s and 1s as electrical/optical signals

Encapsulation Process

Sender: Data → Segment → Packet → Frame → Bits (adding headers at each step)
Receiver: Bits → Frame → Packet → Segment → Data (removing headers at each step)

Infrastructure & Implementation

  • Routers operate at Layer 3 — they read IP packet headers and forward them
  • Switches operate at Layer 2 — they read MAC addresses in frames
  • Hubs/Repeaters operate at Layer 1 — they simply amplify bits
  • Firewalls can operate at multiple layers, filtering frames, packets, or application data
  • PDU inspection is used in Deep Packet Inspection (DPI) for security and QoS

Transmission Impairment

Signals travelling through media are subject to impairment — they degrade or become distorted. Three main types of impairment affect data communication:

1. Attenuation

The loss of energy as a signal travels through a medium. The further it travels, the weaker it gets. Solved with amplifiers (analog) and repeaters (digital).
dB = 10 × log₁₀(P₂/P₁) Attenuation measured in decibels (dB). Negative dB = signal loss.

2. Distortion

Different frequency components of a signal travel at different speeds through a medium, causing the signal shape to change. Common in composite signals. Equalisation circuits compensate for distortion.

3. Noise

Noise TypeCause
Thermal NoiseRandom motion of electrons; present in all electronic devices
Induced NoiseMotors, appliances — external electromagnetic fields
CrosstalkOne wire picks up signal from an adjacent wire
Impulse NoiseSudden spikes — lightning, power surges
SNR = Signal Power / Noise Power Signal-to-Noise Ratio — higher is better
SNRdB = 10 × log₁₀(SNR) SNR expressed in decibels

Geostationary Satellites (GEO)

A geostationary satellite orbits Earth at exactly 35,786 km above the equator, travelling at the same angular speed as Earth's rotation — making it appear stationary from the ground.

The Internet

The Internet is a global network of interconnected computers and networks that communicate using the TCP/IP protocol suite. It evolved from ARPANET (1969) into today's worldwide infrastructure.

Key Internet Concepts

  • IP Address — unique numerical identifier for every device on the internet (IPv4: 32-bit, IPv6: 128-bit)
  • Domain Name System (DNS) — translates human-readable names (google.com) to IP addresses
  • Routing — process of forwarding packets across multiple networks using routing protocols (OSPF, BGP)
  • ISP (Internet Service Provider) — companies providing internet access (Tier 1, 2, 3 hierarchy)
  • World Wide Web (WWW) — hyperlinked documents served over HTTP; invented by Tim Berners-Lee (1989)

Internet Infrastructure Layers

Network Address Translation (NAT)

NAT is a method that allows multiple devices on a private network to share a single public IP address. It was developed as a solution to IPv4 address exhaustion.

Types of NAT

TypeDescription
Static NATOne private IP maps permanently to one public IP (1:1)
Dynamic NATPool of public IPs; assigned temporarily to private IPs
PAT / NAT OverloadMany private IPs share one public IP using different port numbers — most common in homes
Benefits of NAT: Conserves IPv4 addresses · Provides security (hides internal topology) · Allows private IP reuse across organisations

Power over Ethernet (PoE)

PoE is a technology that delivers electrical power alongside data over a standard Ethernet cable (Cat5e or higher). It eliminates the need for separate power supplies for network devices.

How PoE Works

  • A PoE Switch (PSE — Power Sourcing Equipment) injects DC power into the Ethernet cable
  • The powered device (PD) extracts the power from the cable and uses it to operate
  • Power is delivered over the unused wire pairs (Mode A) or all wire pairs (Mode B)
  • Negotiation happens via IEEE 802.3af / 802.3at / 802.3bt standards

PoE Standards Comparison

StandardMax PowerCommon Use
IEEE 802.3af (PoE)15.4 WIP phones, basic cameras
IEEE 802.3at (PoE+)30 WPTZ cameras, video phones
IEEE 802.3bt (PoE++)60 W / 100 WLaptops, TVs, LED lighting
Common PoE-powered devices: IP surveillance cameras · VoIP phones · Wireless access points · Smart lighting · IoT sensors · Thin clients
